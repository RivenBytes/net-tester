این اسکریپت برای تست پورت‌ها، پروتکل‌ها و کیفیت شبکه بین دو سرور طراحی شده است.
-
مثلا شاید بخواید بدونید اصلا ارزش داره برای دوتا سروری که روش هستد وقت بزارید و انواع تانل رو روشون تست کنید؟؟ کد و روی جفت سرور ها میزنید و خیلی راحت از ریورس خوردن نخوردن پروتکل هایی که احتمال جواب میده و چیز هایی که اصلا جواب نمیده مطلع میشید
#
## نحوه استفاده:
اول حتمی مطمعن بشید پورت های 9000 , 5201 روی دو طرف سرور ها باز باشه، محض اطمینان می تونید دستورات زیر رو بزنید:

ufw allow 5201

ufw allow 9000
  
**برای اجرا روی هر دو سرور ایران و خارج، دستور زیر را وارد کنید**:

شماره 2 برای سروری که می خواید به سمتش تست انجام بشه (بهتره اول این اماده بشه و باید روی گزینه 2 بمونه، یعنی وقتی زدید 2 بعدش کنترل c نزنید، صبر کنید تست ارتباطی 1 تموم بشه)

شماره 1 برای سروری که می خواد تست بگیره و باید بهش ip سرور دیگه رو بدید

(بهتر روی دو طرف انجام بشه، یک بار سرور A به B، یک بار هم محض تست ریورس B به A)


```bash
bash <(curl -Ls https://raw.githubusercontent.com/A-battousai/net-tester/main/net-tester.sh)
```
(از سمت سرور دوم هم خواستید تست بیگرید، (مثلا میگیم خارج المان) دو طرف دستور رو بزنید و توی ایران گزینه 2 رو بزنید و بعد سمت خارج گزینه 1 رو بزنید و ip ایران و بدید ..)
